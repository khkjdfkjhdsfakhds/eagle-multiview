# Eagle MultiView

Eagle 4 的安全多窗口伴侣。它不会启动第二个 Eagle 后台，也不会直接写入 `.library` 中的 `metadata.json`；所有修改都经由正在运行的 Eagle Web API 保存。

## 使用

1. 先启动 Eagle 4.0 Build 21 或更高版本并打开目标资料库。
2. 启动 Eagle MultiView。
3. 点击右上角“＋窗口”或按 `Command-N` 创建任意数量的独立窗口。
4. 每个窗口可以停留在不同文件夹、搜索或筛选结果中。

## 文件夹浏览

- 左侧是按父子关系嵌套的真实文件夹树，每个新窗口默认全部折叠。
- “资料库根目录”显示一级文件夹和未归类的根目录文件；“全部素材”才显示全库素材。
- 打开一个文件夹时，中央只显示它的直属子文件夹和直属文件，不会混入更深层后代素材。
- 文件夹卡片带封面、直属文件数和直属子文件夹数；双击或选中后按 `Return` 进入。
- 工具栏支持返回、前进、上一级和可点击的路径导航；`Option-←/→/↑` 也可使用。
- `/` 与 Eagle 原版一致：根据当前层级在“全部展开”和“全部收起”间切换；展开时会滚动定位并高亮当前文件夹。
- 方向键可以在文件夹和素材卡片间移动；`Space`/`Return` 预览素材，`Command-O` 用系统默认应用打开。
- TXT 可直接预览和编辑，`Command-S` 保存；图片、视频、音频、PDF 等可直接预览，PSD 等使用 Eagle 缩略图。
- 右键素材可用默认应用打开、在 Finder 中显示、复制路径/名称或移入废纸篓。
- `Command-Shift-L` 和 `Command-Shift-I` 可分别收起文件夹栏与检查器。

## 编辑与同步

- 支持名称、标签、评分、备注和来源地址。
- 支持把素材拖到左侧文件夹归类、批量添加标签、当前文件夹内置顶和移入废纸篓。
- 导入支持按钮或 `Command-Shift-O` 文件选择、拖入中央素材区、从 Finder 复制文件后在素材区按 `Command-V`，以及粘贴剪贴板图片；普通文件夹视图会导入到当前文件夹，其他视图导入为未分类。
- 标签编辑器使用可添加、粘贴和逐个删除的标签块，支持逗号、中文逗号或 `Return` 快速添加。
- MultiView 内的写入按素材逐项排队；一个窗口保存后会立刻推送到其他窗口。
- 保存前会重新读取 Eagle 中的最新数据。如果另一窗口修改了同一字段，会提示选择载入最新内容或明确覆盖。
- 切换素材、目录或关闭窗口前会拦截尚未保存的检查器修改，避免静默丢失。
- 红色关闭按钮、`Command-W` 与 `Command-Q` 都会显示明确的未保存确认，不会出现窗口无提示地关不掉。
- 标签与文件夹的批量增删采用合并操作，不会用旧数组覆盖另一窗口刚加入的值。
- TXT 保存前会检查外部修改，并先在 MultiView 的应用数据目录保留备份，再以原子替换写入；保存结果会立即通知其他 MultiView 窗口。
- Eagle 原生窗口发生的修改会被自动检测；资料库变化通常约 1 秒触发刷新，可见素材另有短周期复核。

## 安全边界

- Eagle 必须保持运行；断开连接时编辑功能会自动禁用。
- 不提供永久删除，移入废纸篓后仍可在 Eagle 中恢复。
- MultiView 置顶保存在自己的窗口共享状态中，不写 Eagle 元数据；它会识别 Eagle 已有置顶，但 MultiView 内的新置顶只影响 MultiView 的排序。
- 不复制 Eagle 插件、资料库修复、合并等原生管理功能。
- Eagle 切换资料库时，所有 MultiView 窗口会统一跟随，避免跨资料库误写。
- 切换资料库会清空旧库的素材、缩略图和窗口监听缓存；排队中的写入也会在落盘前再次核对资料库路径。
